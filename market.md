# NOVIX 推广文案合集 (Solo Developer Edition)

## 0. 核心定位调整
- **身份设定**：独立开发者，同时也是一名网文/同人写手。
- **痛点切入**：市面上的 AI 写作工具大多是 "Chat" 聊天框模式，写短篇还行，写长篇必崩（记不住设定、逻辑不通）。
- **产品定义**：不是简单的 "AI 生成器"，而是一个 **"本地化、多智能体协作的长篇小说工作台"**。
- **适用人群**：原创网文作者（构建庞大世界观）+ 同人二创作者（需要引入原作设定）。

## 2. 百度贴吧 (Tieba)
**目标吧**：网文吧、起点吧、技术宅吧、Python吧
**风格**：硬核、诚恳、甚至带点"求轻喷"的极客感，强调"解决问题"。

**标题**：【开源】嫌市面上的AI只能写短篇？我手搓了一个"多Agent协作"的本地小说工作台

**正文**：
各位老哥好。
楼主是个写代码的，平时也爱瞎写点玄幻和同人。
**痛点**：大家应该都有体会，现在的 AI（哪怕是 GPT-4），写个 2000 字还行。一旦拉长到 20 章以上，它就开始胡言乱语，人名搞混，战力崩坏，仿佛失忆。

**思考**：这本质上不是模型笨，是**架构**不对。Chat 模式不适合长篇，长篇需要的是**严谨的工程化管理**。

**成果**：
所以我肝了这个 **NOVIX**。我不把它定义为"生成器"，我愿称之为 **"长篇小说工控系统"**。

**核心创新点（区别于套壳站）：**

1.  **拒绝单打独斗**：我设计了 4 个 Agent（主笔、审校、编辑、档案）。
    *   楼主最骄傲的是加入了 **Reviewer (审校)** 角色。它不负责写，只负责**喷**。它会根据设定卡片，检查主笔写的内容有没有违背设定。
    *   *例子：设定里主角"右臂有旧伤"，如果主笔写"主角右手猛击"，审校会直接报错拦截。*

2.  **结构化记忆 (RAG)**：
    引入了 Character Cards 系统。不管是你自己原创的"修仙境界体系"，还是同人的"原作人设"，都能变成结构化数据。
    *   **原创福利**：支持复杂的实体关系（A是B的杀父仇人），AI 生成剧情时会自动检索这段关系。
    *   **同人福利**：**已实装 Wiki 批量提取**。把萌娘百科/Fandom 链接扔进去，自动把网页变成卡片。

3.  **UI/UX**：
    没有花里胡哨的聊天气泡。采用了类似 IDE 的双栏布局，左边写文，右边实时监控 Agent 的思考过程和记忆调用。沉浸式，不干扰。

4.  **完全掌控**：
    代码全开源。数据在本地 json。推荐接 DeepSeek API（性价比之王），或者跑本地 Ollama。不用担心平台倒闭或者审核吞稿。

**碎碎念**：
项目还处于早期，肯定有 Bug。通过这个项目，我只是想证明：**AI 辅助写作，不应该是让 AI 替你思考，而是让 AI 帮你记住那个庞大的世界。**

## 3. 知乎 (Zhihu)
**风格**：技术复盘、产品哲学、对"人机协作"未来的思考。

**问题**：如何评价现在的 AI 写作工具？/ 独立开发者如何利用 Agent 技术落地应用？

**回答**：
谢邀。我是 NOVIX 的开发者。

我在做这个开源项目时，一直在思考一个问题：**为什么现在的 AI 写作工具，很难被专业作者接受？**

我认为核心矛盾在于：**控制权 (Control) vs. 随机性 (Randomness)**。
目前的 Chat 类产品过度放大了随机性，忽视了长篇创作最需要的**一致性控制**。

作为一个独立开发者，我尝试用 **Agentic Workflow (智能体工作流)** 来解决这个问题。 NOVIX 就是我的答卷。

**我的设计哲学：**

1.  **去 Chat 化 (De-Chat interface)**
    写作不是聊天。NOVIX 的界面更像是一个 Scrivener 或 IDE。所有的交互是围绕"项目"和"卡片"展开的，而不是流式对话。

2.  **制衡机制 (Check and Balance)**
    在 NOVIX 里，"写"只是环节之一。我引入了 `Reviewer Agent`。在传统的 LLM 交互中，User 是唯一的 Reviewer。但在 NOVIX 里，我把"设定集"赋予了 Reviewer，让 LLM 自己监督自己。
    *   这对于**原创作者**意味着：你的世界观不再是摆设，而是真正生效的约束力。
    *   这对于**同人作者**意味着：OOC 的概率被系统性地降低了。

3.  **记忆的"工程化"**
    人类作者靠脑子和烂笔头记设定。NOVIX 靠 `Archivist Agent`。
    为了降低门槛，我刚刚上线了 **Wiki-to-Card 批量提取引擎**。这是利用 LLM 的语义理解能力，将非结构化的网页（Wiki）转化为结构化的知识图谱。这不仅是为了同人，也是为了让作者能快速把灵感素材库转化为 AI 可读的记忆。

**写给开发者和创作者：**
NOVIX 完全开源及本地化。我不希望它成为一个割韭菜的 SaaS 工具，而希望它是一个像 VS Code 一样的基础设施。
无论你是想构建自己克苏鲁神话体系的原创作者，还是想为《鸣潮》写百万字长篇的同人大大，NOVIX 都在尝试为你提供一个"除了灵魂，其他都能帮你搞定"的虚拟助手。
